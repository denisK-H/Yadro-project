# Автоматизация измерения АЧХ усилительного каскада

Выполненная работа: программный комплекс для управления генератором сигналов и осциллографом, автоматического съема точек АЧХ и поиска частоты единичного усиления.

---

## Цель работы

- Автоматизировать лабораторное измерение амплитудно-частотной характеристики.
- Исключить ручной перебор частот и ручную запись значений.
- Получать воспроизводимые данные в формате CSV.
- Автоматически находить область, где коэффициент усиления становится равным единице.
- Строить графики по результатам измерений.

---

## Используемые приборы

- Генератор сигналов: формирует синусоидальный входной сигнал.
- Осциллограф OWON XDS3064E: измеряет входное и выходное напряжение.
- Канал CH1: входной сигнал, Vin.
- Канал CH2: выходной сигнал, Vout.
- Связь с приборами выполняется по USB через VISA/SCPI.

```text
config.yaml

    |
    | HTTP REST API
    v

    |

    v
Генератор сигналов и осциллограф
    |
    v
results.csv -> plot.py -> графики
```

---

## Роль Go-программы

- Читает параметры эксперимента из `config.yaml`.
- Подключается к REST API моста.
- Отправляет SCPI-команды для настройки приборов.
- Управляет частотным планом эксперимента.
- Запрашивает измеренные значения Vin и Vout.
- Рассчитывает коэффициент усиления:

```text
Ku = Vout / Vin
Ku_dB = 20 * log10(Ku)
```

- Сохраняет результат в `results.csv`.

## Роль Python REST API

Файл `generator.py` работает как локальный мост между Go-программой и измерительными приборами.

Основные REST-методы:

- `/connect` - подключение к прибору по VISA resource.
- `/write` - отправка SCPI-команды без чтения ответа.
- `/query` - отправка команды и чтение текстового ответа.
- `/query_number` - чтение числового результата измерения.
- `/close_all` - закрытие соединений с приборами.

## Настройка из config.yaml

В конфигурации задаются:

- адрес REST API;
- VISA-адреса генератора и осциллографа;
- начальные SCPI-команды;
- диапазон частот;
- задержки перед чтением;
- число повторов при ошибках;
- параметры ручного масштабирования осциллографа;
- параметры поиска частоты единичного усиления.

Такой подход позволяет менять методику без изменения исходного кода.

## Этапы работы программы

1. Чтение `config.yaml`.
2. Подключение генератора и осциллографа.
3. Настройка осциллографа: включение CH1/CH2, режим AC, нулевое смещение.
4. Настройка генератора: синусоидальный сигнал, заданная амплитуда, включение выхода.
5. Выбор очередной частоты.
6. Настройка временного масштаба осциллографа.
7. Ручная подстройка вертикального масштаба.
8. Измерение Vin и Vout.
9. Расчет Ku и Ku в дБ.
10. Запись точки в CSV.

## Методика измерений

На каждой частоте генератор устанавливает синусоидальный сигнал, а осциллограф измеряет размах напряжения на входе и выходе.

Измеряются:

- `Vin` - амплитуда входного сигнала, Vpp;
- `Vout` - амплитуда выходного сигнала, Vpp;
- `Ku` - коэффициент усиления;
- `Ku_dB` - коэффициент усиления в децибелах.

Для повышения устойчивости добавлены задержки установления сигнала и повторные попытки измерения.

---

## Масштабирование осциллографа

Автоматический autoscale осциллографа не используется, потому что он может обрезать часть сигнала.

Вместо этого реализована ручная логика:

- выбирается временная развертка так, чтобы на экране помещалось несколько периодов;
- вертикальный масштаб CH1 и CH2 подбирается по измеренному Vpp;
- если прибор возвращает некорректное значение или сигнал не помещается, масштаб увеличивается;
- смещение каналов выставляется в ноль.

Это делает измерения стабильнее и уменьшает число ошибок вида `OFF` или `?`.

## Поиск частоты единичного усиления

Частота единичного усиления соответствует условию:

```text
Ku = 1
Ku_dB = 0 dB
```

Поиск выполняется по логарифмической сетке частот. Если переход через Ku=1 найден между двумя точками, программа уточняет результат дополнительными измерениями внутри этого интервала.

Диапазон поиска задается в `unity_search`:

- стартовая частота;
- максимальная частота плана;
- жесткий верхний предел;
- число точек на декаду;
- допустимая погрешность.

---

## Формат результата

Основной файл результата: `results.csv`.

Столбцы:

- `frequency_hz` - частота;
- `vin_vpp` - входное напряжение;
- `vout_vpp` - выходное напряжение;
- `gain` - линейный коэффициент усиления;
- `gain_db` - усиление в дБ;
- `ok` - успешность измерения;
- `point_type` - тип точки;
- `vin_raw`, `vout_raw` - исходные ответы осциллографа.

---

## Построение графиков

Файл `plot.py` строит графики по `results.csv`:

- `ach_linear.png` - линейный коэффициент усиления от частоты;
- `ach_db.png` - АЧХ в децибелах;
- `vin_vout.png` - входное и выходное напряжение от частоты.

На графике в дБ дополнительно отмечается уровень 0 dB и найденная область единичного усиления.

## Пример работы

По одному из запусков было получено 48 измеренных точек.

Пример начальной точки:

```text
f = 1000 Hz
Vin = 0.260 Vpp
Vout = 1.120 Vpp
Ku = 4.31
Ku_dB = 12.68 dB
```

В области единичного усиления:

```text
f = 4.6639 MHz, Ku = 1.038, Ku_dB = 0.322 dB
f = 4.8919 MHz, Ku = 0.943, Ku_dB = -0.506 dB
```

Значит, частота единичного усиления находится примерно между 4.66 и 4.89 МГц.
